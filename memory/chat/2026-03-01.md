## VMs vs Containers for Agents

**Containers are the right default** for most agent use cases:
- Fast spawn (ms vs minutes)
- Lower overhead (shared kernel)
- Good enough isolation for trusted agents
- Ecosystem (Docker Hub, compose, etc.)

**VMs make sense when:**
- Multi-tenant isolation (agents from different orgs)
- Running truly untrusted code at scale
- Need different OS kernels
- Regulatory compliance (hardware-level isolation)

**The elegant middle ground:** MicroVMs
- **Firecracker** (AWS Lambda) - VM isolation, container speed
- **Kata Containers** - OCI-compatible VMs
- **gVisor** - syscall-level sandbox (slower but stronger)

For Claw Pen, I'd suggest: **containers by default, optional MicroVM mode** for high-security scenarios.

---

## Massive Codebases

The current volume mount approach doesn't scale well for:
- 100k+ files
- Multi-GB repos
- Cross-machine latency (macOS/Windows → Linux container)

**Elegant solutions already exist:**

| Approach | Best For | Trade-off |
|----------|----------|-----------|
| **Lazy loading** | Large repos | Agent only sees what it requests |
| **Semantic index** | Code search | Pre-index, query fast |
| **Remote workspace** | Huge projects | Code stays on server, no sync |
| **Syncthing/Mutagen** | Cross-platform | Background sync, faster than mount |
| **NFS/SMB mount** | Shared storage | Network latency |

**My recommendation for Claw Pen:**

1. **Small/medium projects** → Volume mount (current)
2. **Large codebases** → Agent runs *on* the code server (remote mode)
3. **Code search** → sqlite-vss or Qdrant for semantic indexing

The "elegant" solution already exists: **agents should run where the data lives**, not ship data to the agent. For your Claw Pen architecture, that means:

- **Local agents** → Volume mounts, good for <10GB
- **Remote agents** → Deploy to the server with the data
- **Hybrid** → Semantic index locally, fetch files on demand

## Safe Agent Distribution Model

```
┌─────────────────────────────────────────────────────────┐
│                    USER'S MACHINE                        │
│                                                          │
│  ┌──────────────┐     ┌──────────────────────────────┐  │
│  │  Large Repo  │◄────│  Agent Container (local)     │  │
│  │  ~/projects  │     │  - Template from ClawHub     │  │
│  │  50GB        │     │  - Runs in sandbox           │  │
│  └──────────────┘     │  - Network: disabled/allowlist│  │
│                       └──────────────────────────────┘  │
│                                  │                       │
│                                  ▼                       │
│                         Only talks to                    │
│                         local LLM API                    │
└─────────────────────────────────────────────────────────┘
```

**Agent template is tiny** (few MB) - just code + config
**Data stays local** - never leaves user's machine
**User controls access** - chooses which folders to mount

## How It Works

1. **User downloads agent template** from ClawHub (signed, verified)
2. **User specifies what to share:**
   ```yaml
   folders:
     - ~/projects/my-app:/workspace:ro  # read-only
     - ~/projects/my-app/src:/workspace/src:rw  # write access
   network: none  # or allowlist: ["api.openai.com"]
   ```
3. **Agent runs locally** with only that access
4. **No exfiltration possible** - network blocked, only local filesystem

## For Huge Codebases

**Option 1: Lazy Index (recommended)**
```yaml
folders:
  - ~/huge-repo:/workspace:ro
  
index:
  type: semantic  # sqlite-vss
  paths:
    - /workspace/**/*.py
    - /workspace/**/*.rs
  # Agent builds index on first run, queries fast after
```

Agent:
- Scans and indexes codebase on first run
- Stores semantic index locally
- Queries index to find relevant files
- Only reads what it needs

**Option 2: Shared Volume Pool**
```yaml
# User defines shared data pools
pools:
  work-code:
    path: ~/work-projects
    size: 500GB
    access: read-only
    
# Agent mounts from pool
agent:
  mounts:
    - pool: work-code
      at: /workspace
```

Multiple agents can share the same pool - no duplication.

## Security Model

| Layer | Protection |
|-------|------------|
| **Template** | Signed, hash-verified from ClawHub |
| **Runtime** | Container/namespace isolation |
| **Filesystem** | Only mounted paths visible |
| **Network** | Disabled by default, explicit allowlist |
| **Secrets** | Never in template, injected at runtime |

## Implementation for Claw Pen

```rust
// Agent creation with safe folder mounting
CreateAgent {
    template: "code-assistant",
    mounts: vec![
        Mount::new("~/my-project", "/workspace", Access::ReadOnly),
        Mount::new("~/my-project/src", "/workspace/src", Access::ReadWrite),
    ],
    network: NetworkPolicy::None,  // or Allowlist(["localhost:11434"])
    secrets: HashMap::new(),  // injected, not in template
}
```

**Key principle:** The agent template is public and portable. The data and secrets are private and local. They meet at runtime on the user's machine.

// === Mount Configuration ===

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct Mount {
    /// Host path (supports ~ expansion)
    pub host_path: String,
    /// Container path
    pub container_path: String,
    /// Access mode
    #[serde(default)]
    pub access: MountAccess,
}

#[derive(Debug, Clone, Serialize, Deserialize, Default)]
#[serde(rename_all = "lowercase")]
pub enum MountAccess {
    #[default]
    ReadOnly,   // ro
    ReadWrite,  // rw
    NoExec,     // ro,noexec - for sensitive data
}

impl Mount {
    pub fn new(host: &str, container: &str, access: MountAccess) -> Self {
        Self {
            host_path: host.to_string(),
            container_path: container.to_string(),
            access,
        }
    }
    
    /// Expand ~ to home directory
    pub fn expand_host_path(&self) -> String {
        if self.host_path.starts_with('~') {
            if let Some(home) = std::env::var("HOME").ok() {
                return self.host_path.replacen('~', &home, 1);
            }
        }
        self.host_path.clone()
    }
    
    /// Convert to Docker mount string
    pub fn to_docker_mount(&self) -> String {
        let mode = match &self.access {
            MountAccess::ReadOnly => "ro",
            MountAccess::ReadWrite => "rw",
            MountAccess::NoExec => "ro,noexec",
        };
        format!("{}:{}:{}", self.expand_host_path(), self.container_path, mode)
    }
}

// === Network Policy ===

#[derive(Debug, Clone, Serialize, Deserialize, Default)]
pub struct NetworkPolicy {
    /// Network mode
    #[serde(default)]
    pub mode: NetworkMode,
    /// Explicit allowlist (only used in Allowlist mode)
    #[serde(default)]
    pub allowlist: Vec<String>,
    /// DNS servers to use (optional override)
    #[serde(default)]
    pub dns: Vec<String>,
}

#[derive(Debug, Clone, Serialize, Deserialize, Default)]
#[serde(rename_all = "lowercase")]
pub enum NetworkMode {
    #[default]
    None,       // No network access at all
    Local,      // Only localhost (for LLM APIs)
    Allowlist,  // Only specific hosts
    Bridge,     // Full network (default Docker behavior)
}

impl NetworkPolicy {
    /// No network access - safest for code analysis
    pub fn none() -> Self {
        Self {
            mode: NetworkMode::None,
            allowlist: vec![],
            dns: vec![],
        }
    }
    
    /// Local only - can talk to localhost services
    pub fn local() -> Self {
        Self {
            mode: NetworkMode::Local,
            allowlist: vec!["127.0.0.1".to_string(), "::1".to_string()],
            dns: vec![],
        }
    }
    
    /// Allowlist with specific hosts
    pub fn allowlist(hosts: Vec<&str>) -> Self {
        Self {
            mode: NetworkMode::Allowlist,
            allowlist: hosts.iter().map(|s| s.to_string()).collect(),
            dns: vec![],
        }
    }
    
    /// Convert to Docker network config
    pub fn to_docker_network(&self) -> Option<String> {
        match &self.mode {
            NetworkMode::None => Some("none".to_string()),
            NetworkMode::Local | NetworkMode::Allowlist => None, // Will use custom iptables
            NetworkMode::Bridge => Some("bridge".to_string()),
        }
    }
    
    /// Generate iptables rules for allowlist mode
    pub fn to_iptables_rules(&self) -> Vec<String> {
        if self.mode != NetworkMode::Allowlist {
            return vec![];
        }
        
        let mut rules = vec![
            // Drop all outgoing by default
            "iptables -A OUTPUT -j DROP".to_string(),
        ];
        
        // Allow localhost
        rules.push("iptables -A OUTPUT -d 127.0.0.0/8 -j ACCEPT".to_string());
        
        // Allow specific hosts
        for host in &self.allowlist {
            rules.push(format!("iptables -A OUTPUT -d {} -j ACCEPT", host));
        }
        
        rules
    }
}
